# Support

## Before opening an issue

- read the [README](./README.md)
- check the [CLI install guide](./docs/cli/install.md)
- check the [5-minute quickstart](./docs/cli/quickstart.md)
- check the [CLI reference](./docs/cli/commands.md)
- if you need the Maestro CLI map for agents, open the generated `AGENTS.md` in the workspace root
- inspect the versioned examples in [`examples/`](./examples/)
- run `pnpm check` if you are working locally

## Where to ask what

- use GitHub Issues for reproducible bugs, documentation gaps, and concrete feature requests
- use pull requests for proposed fixes with tests or documentation updates
- use security reporting channels for vulnerabilities instead of public issues

## What helps maintainers most

- a minimal reproduction
- the workspace or pack shape involved
- the command you ran
- the observed result and the expected result
